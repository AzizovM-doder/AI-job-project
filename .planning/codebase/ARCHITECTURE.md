# System Architecture

This document describes the high-level architecture and design patterns used in the AI-JOB/ai codebase.

**Last Updated:** 2026-04-19

## Overall Pattern

The project is built on **Next.js 16**, following the **App Router** paradigm. It emphasizes a mix of Server Components for performance and SEO, and Client Components for rich interactivity.

### Core Architectural Principles
- **Modern React**: Utilizing React 19 features (Server Components, Actions, latest hooks).
- **Type Safety**: Unified TypeScript types across components, store, and API layers.
- **Atomic-ish Development**: UI components are separated into a reusable `ui/` directory (Shadcn), while business logic components are feature-bound.
- **Glassmorphism / Premium UI**: Aesthetic choices prioritize transparency, blur effects, and high-quality motion (Framer Motion).

## Routing & Rendering

- **App Router**: Routing is defined by the file system under `src/app/`.
- **i18n Support**: A top-level `[locale]` segment handles internationalization (e.g., `/en/dashboard`, `/az/dashboard`).
- **Layouts**: Nested layouts are used extensively to maintain navigation and state across page transitions.

## Data Flow

### Server Data
- Initial page data is often fetched in Server Components or passed via layouts.
- API interactions for dynamic data use **TanStack React Query**.

### Client Data
- **React Query Hooks**: Generated by Orval, these manage server-state fetching, caching, and mutations.
- **Zustand Stores**: Global client-side state (e.g., auth session, UI preferences, volatile settings) is centralized in `src/store/`.

## Styling Strategy

- **Tailwind CSS 4**: The project uses the latest Tailwind CSS for utility-first styling.
- **Component-Level Styling**: Complex components use `class-variance-authority` (CVA) to manage variants (e.g., different button sizes or themes).
- **Dynamic Animations**: Framer Motion is the primary engine for layout transitions and micro-interactions.

## Directory Significance

- **`src/@types`**: (Optional) Global type overrides.
- **`src/app`**: Routing, pages, and server-centric logic.
- **`src/components`**: Reusable UI and business components.
- **`src/hooks`**: Custom React hooks, including generated API hooks.
- **`src/lib`**: Utility functions, shared constants, and core client instances (Axios).
- **`src/store`**: Zustand store definitions.
- **`src/types`**: Domain-specific TypeScript interfaces.
